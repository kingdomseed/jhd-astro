# Domain Docs

This is a single-context repo.

Before domain-sensitive work, read root `CONTEXT.md`. If `docs/adr/` exists,
read ADRs relevant to the area being changed.

Use the glossary's vocabulary in issue titles, plans, hypotheses, test names,
and copy proposals. If a needed term is missing or ambiguous, resolve it
through `grill-with-docs` and update `CONTEXT.md` inline.

If an output contradicts an ADR, surface the conflict explicitly rather than
silently overriding it.
