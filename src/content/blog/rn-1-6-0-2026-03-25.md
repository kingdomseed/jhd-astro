Released on March 25, 2026

This is the biggest feature release since the app launched. Three systems from the Mythic GME 2e rules that players have been requesting — Progress Tracks, Discovery Checks, and Keyed Scenes — all landed together. The Game Panel got a full redesign to hold them: it's now a tabbed layout with Meaning Tables, Event Focus, Tracks, and Discovery each in their own tab.

I've been working on these for a while (the Discord #daily-wrap has the receipts), and each one went through a few design iterations before I was happy with how they felt in play. Here's what's in the box.

## Highlights

- **Progress Tracks**: Visual progress bars for tracking thread advancement. Set custom point ranges, mark Flashpoints in the Adventure Log, and watch the bar fill as the story moves forward. Controls use a two-tap "armed action" pattern — tap once to arm, tap again to confirm — so you don't accidentally bump your progress.
- **Discovery Checks**: A dedicated panel for the oracle-style Discovery Check mechanic. Roll, get an automated result summary, and (if you have Progress Tracks active) the result auto-applies to the right track. Localized across all 8 languages.
- **Keyed Scenes**: Set up scene triggers that fire when conditions are met. Choose count-based targets with a visual stepper, pick one-shot (deactivates after trigger) or recurring (resets and stays active), and let the app auto-trigger when the count reaches the target. Triggered scenes get a badge and a dedicated log entry.
- **Tabbed Game Panel**: The old single-panel view is gone. Meaning Tables, Event Focus, Tracks, and Discovery now live in their own tabs — less scrolling, faster switching.

## Added

- **Progress Tracks**: Thread progress tracking with visual progress bars, custom point ranges, and Flashpoint log entry support.
- **Discovery Checks**: Dedicated Discovery Check panel with oracle-style outcomes, automated result summaries, and localized strings.
- **Keyed Scenes**: Triggered scene badges, inline auto-increment toggles, count-based targets (one-shot and recurring), and dedicated game log builders.
- **Game Panel**: Refactored into a tabbed layout housing Meaning Tables, Event Focus, Tracks, and Discovery.
- **Architecture**: Lifecycle guards on all major notifiers (Journal, Table, Formula, Event Focus) to harden iCloud operations against app backgrounding.

## Changed

- Updated Flutter SDK to 3.41.5 across all build tooling.
- Progress Track controls use +1/+2/+3 increments with a new FP (Focus Point) modifier button.
- Regenerated iOS Xcode project with updated CocoaPods references for build consistency.

## Fixed

- Progress Track controls now wrap properly on narrow panels (no more layout overflow).
- Improved iOS journal file watching reliability and compact empty state rendering.
